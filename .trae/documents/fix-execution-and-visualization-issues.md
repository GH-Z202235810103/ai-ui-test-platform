# 修复测试执行失败和可视化图表显示问题

## 问题分析

### 1. 测试执行失败的根本原因
- **位置**: `backend/core/executor_optimized.py` 第377行
- **问题**: `step_desc[:20]` 直接对步骤描述进行切片，但 `step_desc` 可能是字典类型
- **错误**: `TypeError: 'slice' object is not subscriptable` 或类似错误
- **影响**: 导致测试执行中断，无法保存报告到数据库

### 2. 可视化图表没有数据的原因
- **直接原因**: 测试执行失败导致报告没有保存到数据库
- **间接原因**: 
  - 数据库查询可能返回空结果
  - API 端点数据处理逻辑可能有问题
  - 前端组件可能没有正确处理空数据状态

### 3. 代码逻辑问题
- 在 `_execute_step` 方法中，第253行提取了步骤文本，但在第377行没有使用
- 截图命名时应该使用提取后的文本，而不是原始的 `step_desc`

## 技术细节

### 修复 slice 错误的具体代码

**当前代码** (第377行):
```python
screenshot_name = self._take_screenshot(driver, None, step_idx, step_desc[:20])
```

**修复后的代码**:
```python
# 在方法开始处提取步骤文本
step_text = step_desc if isinstance(step_desc, str) else step_desc.get('description', str(step_desc))

# 在截图命名时使用提取后的文本
safe_step_text = re.sub(r'[^\w\-]', '', step_text)[:20]
screenshot_name = self._take_screenshot(driver, None, step_idx, safe_step_text)
```
