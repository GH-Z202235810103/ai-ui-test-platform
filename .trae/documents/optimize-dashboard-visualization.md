# 优化仪表盘可视化图表方案

## 问题分析

### 当前仪表盘结构
1. **第一行**：4个统计卡片（项目总数、测试用例、测试任务、测试报告）
2. **第二行**：任务状态分布 + 快速操作
3. **第三行**：测试执行趋势图（TrendChart）
4. **第四行**：整体通过率环形图（PassRateRing）+ 最近测试活动时间线（ActivityTimeline）

### 存在的问题
1. **内容重复**：任务状态分布和整体通过率环形图都展示了测试状态信息
2. **空间利用不合理**：ActivityTimeline 组件占用大量空间但信息密度低
3. **可视化效果不佳**：部分组件显示为空或数据不足时没有友好的提示
4. **功能重叠**：多个组件展示相似的状态信息，缺乏多样性

## 优化方案

### 新的仪表盘布局

#### 第一行：统计卡片（保持不变）
- 项目总数
- 测试用例
- 测试任务
- 测试报告

#### 第二行：核心可视化图表
**左侧（span=16）**：测试执行趋势图（TrendChart）
- 展示最近7天/30天的测试执行趋势
- 包含执行次数、通过率、失败用例等多维度数据
- 保留原有的时间范围切换功能

**右侧（span=8）**：测试状态分布饼图（新增）
- 替换原有的"任务状态分布"和"整体通过率环形图"
- 使用饼图展示测试用例的状态分布（通过、失败、待执行、运行中）
- 点击扇区可以跳转到对应的用例列表
- 显示总数和百分比

#### 第三行：辅助信息展示
**左侧（span=12）**：测试用例类型分布（新增）
- 使用柱状图展示不同类型测试用例的分布
- 如：UI测试、API测试、录制回放、AI生成等
- 显示每种类型的数量和占比

**右侧（span=12）**：最近执行记录表格（替换 ActivityTimeline）
- 使用表格形式展示最近10条执行记录
- 包含：用例名称、执行时间、状态、耗时、操作按钮
- 更紧凑的布局，信息密度更高
- 支持快速查看详情和重新执行

#### 第四行：快速操作（保持不变）
- 管理项目
- 创建用例
- 新建任务
- 录制回放

## 具体实现步骤

### 任务1：创建测试状态分布饼图组件
**文件**：`frontend/src/components/dashboard/StatusPieChart.vue`

**功能**：
- 使用 ECharts 饼图展示测试状态分布
- 支持点击扇区跳转到对应用例列表
- 显示总数和百分比
- 空数据时显示友好提示

**数据源**：`/api/v2/statistics` 接口返回的 `task_status` 数据

### 任务2：创建测试用例类型分布柱状图组件
**文件**：`frontend/src/components/dashboard/TypeDistribution.vue`

**功能**：
- 使用 ECharts 柱状图展示不同类型测试用例的分布
- 显示每种类型的数量和占比
- 支持点击柱子跳转到对应类型的用例列表
- 空数据时显示友好提示

**数据源**：需要扩展 `/api/v2/statistics` 接口，添加用例类型统计

### 任务3：创建最近执行记录表格组件
**文件**：`frontend/src/components/dashboard/RecentExecutions.vue`

**功能**：
- 使用 Element Plus 表格展示最近10条执行记录
- 列：用例名称、执行时间、状态、耗时、操作
- 支持查看详情和重新执行
- 空数据时显示友好提示

**数据源**：`/api/v2/recent-activities` 接口

### 任务4：更新仪表盘布局
**文件**：`frontend/src/views/Dashboard.vue`

**修改内容**：
1. 移除第二行的"任务状态分布"和"快速操作"卡片
2. 调整第三行布局：TrendChart 占 16 列，StatusPieChart 占 8 列
3. 新增第四行：TypeDistribution 和 RecentExecutions 各占 12 列
4. 保留第五行的"快速操作"卡片

### 任务5：扩展后端API
**文件**：`backend/api/v2/endpoints.py`

**修改内容**：
1. 扩展 `/api/v2/statistics` 接口，添加用例类型统计
2. 确保返回的数据格式符合前端组件的期望

## 预期效果

### 优化后的优势
1. **信息层次清晰**：从概览到详细，层层递进
2. **空间利用合理**：重要信息占据更大空间，次要信息紧凑展示
3. **可视化多样性**：包含趋势图、饼图、柱状图、表格等多种形式
4. **交互友好**：支持点击跳转、快速操作等
5. **数据完整性**：即使某些数据为空，也有友好的提示信息

### 数据流向
```
后端API
  ├─ /api/v2/statistics
  │   ├─ 基础统计（项目、用例、任务、报告）
  │   ├─ 任务状态分布
  │   └─ 用例类型分布（新增）
  ├─ /api/v2/reports/trends/overview
  │   └─ 测试执行趋势
  └─ /api/v2/recent-activities
      └─ 最近执行记录
```

## 风险评估

### 低风险
- 创建新组件：不影响现有功能
- 调整布局：只是视觉调整

### 中风险
- 扩展后端API：需要确保向后兼容
- 数据格式变更：需要测试前端组件的兼容性
